4) RELATIONS IMPORTANTES (lecture rapide)
============================================================
- User est la table centrale (auth + profils + acteur metier).
- Reservation relie User et Space.
- Course relie Instructor(User), Assignments, Sessions, Enrollments.
- TrainingSessionAttendee est le pivot Session<->User.
- TeamMember est le pivot Team<->User.
- SpaceEquipment est le pivot Space<->Equipment.
- AssociationMember est le pivot Association<->User.
- Notification peut pointer vers Reservation/Course/Session pour contexte.

8) COMMENT PARLER CORRECTEMENT A TON APP (API)
============================================================
1) Authentifie toi d abord:
- POST /api/auth/login
- Recupere le token JWT

2) Envoie toujours le token:
- Header: Authorization: Bearer <token>
- Content-Type: application/json

3) Respecte les types:
- IDs en nombre (ex: 12)
- Dates ISO (ex: 2026-04-13T10:00:00.000Z)
- Booleens vrais/faux (true/false)

4) Pagination et filtres:
- skip et take pour paginer
- Exemple notifications: /api/notifications?skip=0&take=20&isRead=false&type=RESERVATION_REMINDER_24H

5) Flow exemple notifications:
- GET /api/notifications/unread-count
- GET /api/notifications
- PATCH /api/notifications/:id/read
- PATCH /api/notifications/read-all
- DELETE /api/notifications/:id

6) En cas d erreur:
- 401: non authentifie
- 403: non autorise
- 404: ressource introuvable
- 422/400: validation des donnees
- 500: erreur serveur
